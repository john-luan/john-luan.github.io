Setup STM32 Development Environment

John's Holic
2019-04-15

在debian和wsl下搭建stm32或者是cortex-m系列的MCU的开发环境。

契子

是做了一个SDR，用stm32来配置transceiver。检查环境的时候想要不写点小作文什么的。

总体来说，这篇文章简述了如何搭建一个代码交叉编辑环境。它支持官方的硬件调试器，通过符合直觉的方式进行常规的在线调试。

行事

将会用到的工具

linux下

  • 编译器 gnu-none-eabi
  • 代码编辑，调试环境 emacs(emacs-gdb) + gnu-none-eabi-gdb + gdbgui
  • 硬件调试工具 openocd + stlink
    关于gdb的使用，建议参考韦易笑的这篇文章

Windows下

  • 调试/烧录工具 cube programmer
  • 运行编译的虚拟环境 WSL
  • 编辑器，调试器 VSCode

需要编写的文件

  • ld文件
  • makefile
  • 一段基础的串口调试代码

环境

debian 9/windows 11
ld 文件和 makefile的分析放到后面专门找个时间来说吧

硬件上的考虑

目标硬件为stm32f412
同时也会准备STM32G4/L4/G0和NXP环境
调试器是st-link v3 mini

简单说两句

  1. 什么是openOCD(Open On-Chip Debugger) - 以某种特定的内存映射和协议格式提供MCU core和调试接口通信的软件，依赖xx-link控制硬件
  2. 什么是CMSIS(Common Microcontroller Software Interface Standard) - Cortex-M处理器系列的与供应商无关的硬件抽象层。主要提供内核寄存器，中断，数据结构等一致性环境

调试器提供的功能

  1. 确保必要的环境已经安装
  2. clone stlink代码仓库
  3. 编译
    cd cpl_stlink
    cmake .
    make
  4. 安装二进制文件到系统目录
    cp bin/st-* /usr/local/bin
    cp lib/*.so* /lib64
  5. 因为安装了新的设备，重新加载udev
    udevadm control –reload-rules && udevadm trigger
