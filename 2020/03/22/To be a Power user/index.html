To be a Power User

如何使用计算机？

最近总是被人科普Windows不好用，macOS才是终极生产力工具balabala……
于是我想写一篇文章说说我是怎么使用Windows(这里的Windows指滚动升级并安装在一个还过得去的硬件上)的，生产力在哪。

利益相关

我是一个果黑(显而易见)，如果问我原因，你会得到这个答案：
“苹果的一些设备和服务的却不错，但在这家公司的逻辑中用户都是盲目无知且愚昧到需要为他们包办一切行为的蠢货；其二，部分极端的果粉太过激进并精神入股了这家公司，这让我十分不适。为了避免和这群人被同时称为‘苹果用户’我没有购买任何苹果的设备和服务”

注意：如果您是一位果粉(甚至比较狂热)，这篇文章极大可能将使您感到不适。
上面的一小段言论中已经表现出我的极端态度。
但显然，我并不是想在这个地方和您争论。
我甚至都没有开放评论，因为这个是我自己的博客，我在这个空间里享有绝对的言论自由。
您应该意识到这一点并为自己接下来的阅读状态负责。

选择软件的原则

尽可能避免臭名昭著的国产管家类软件
因为需求而选择软件，而非因为逼格或者其他东西
不要带有偏见(不管是颜值还是一些潜意识的印象)

为何打开电脑 & 需求

用$$网络$$来打发时间并不是一件好事，这篇文章也着重讨论”use it”而非”play it”。
坐到电脑前，就应该清楚“要做什么” 以及 “用什么工具做什么”。
如果对这个问题感到迷茫，可以试试emacs的org-mode，或者出去走走，运动或者泡咖啡馆什么的。你应当成为一个现充:)

  1. 我同时需要Windows和Linux环境(为什么？试试在macOS下安装一下Cadence Virtuoso的Linux版)
  2. 我需要使用或者搭建云服务在各个设备间同步工程文档

我打开电脑做这么几件事：

电路设计
开发FPGA/ARM/Linux
写文档，写博客
打游戏(特指StarCraftII)
爱好的辅助(三维设计)

我如何使用电脑

  1. 我在Windows下安装虚拟机(WLS & VirtualBox)运行Debian和Gentoo。
    使用虚拟机从根本上杜绝了文件同步，输入法，图形界面，Office，Flash，IE，网银等等等等问题。而且可以同时开启，极其方便的切换。
    正确的做法是把Linux和Windows融为一体，而不是不休止的争论他俩的好坏。
    Linux成为一个字符终端，一个编辑器，一个IDE，一个文件管理工具；而Windows成为一个图形终端，一个浏览器，一个workbench。
    (但是macOS做不到，它只能代替Linux或者与Windows非常尴尬的共存(使用mac不推荐的第三方引导安装双系统或浪费大量(尤其是(并不宽裕的)图形)性能给虚拟机中的Windows))。

使用

我有三个存储区域，分别是
256G SSD TOSHIBA - GPT - 单一分区 - Windows System / Linux Virtual Disk / Softwares on Windows
256G tf_card SAMSUNG - GPT - 单一分区 - Workspace(工程目录) ，该目录下子目录使用前缀进行区分
prj_ 软件项目
brd_ PCB项目
lrn_ 为学习某种技能/语言/工具设置的测试目录(例如lrn_cl = learn common lisp)
red_ 阅读目录，我读书有记笔记的习惯。混合使用Markdown OneNote和Latex
cpl_ 一些需要自己构建/通过Git追新的软件的编译目录(例如vim，我总是编译最新版)

2T Seagate ST1000LM035×3 - GPT - RAID1 - Media目录
这是挂载到NAS的三块HHD组成的RAID1，将Windows图片视频文档下载等皆映射到此。Linux也有rw权限
关于为什么是RAID1: 本人一共用死过5+块硬盘，一块U。可能是脸黑吧……

  • 不安装任何杀毒软件，管家软件。系统自带Defender & 防火墙够用了。因为我开机只做如上几件事。
    感染病毒的可能性很低而且国产管家软件全家桶在我看来的作用仅仅是拖慢电脑运行速度和无限弹窗。
    (这就解释了为什么我的电脑没有弹窗)
  • 系统自带输入法够用了(RIME也很清真)。
  • 开机启动项
    Shadowsocks/v2ray - 我需要使用Google和OneDrive
    OneDrive - 用于多设备同步
    WOX - 快速启动工具
    Snipaste - 截图工具

VirtualBox虚拟机 && WSL && putty

系统装在SSD上，媒体资料存储在NAS(/media)上，开源项目同步到GitHub，私有项目存到works盘并由Debian管理，通过Onedrive同步。
/*

  • 我几乎没什么媒体文件，我不喜欢看剧，不喜欢看电影。更没有保存它们的习惯。
  • 并且我的生活相当无聊，去旅行是因为已经在崩溃的边缘，自然也无心拍照，遑论摄影。
  • /
    不安装im软件，对我来说它们在手机/Chrome上已足够用。(但是我装了个telegram的客户端)
    我几乎可以使用快捷键完成的功能都会使用快捷键(这也就解释了为什么我在除了打游戏和三维制图以外不使用鼠标和触摸板)，没有快捷键的软件，我会尽量使用AHK和软件API使用快捷键。
    我使用Linux(Gentoo)时：

将Gentoo安装在VirtualBox里。虚拟磁盘文件放在SSD分区。
通过SSH(Putty)连接到虚拟机，纯CLI。全屏使用。通过alt+tab来切换”Linux”和”Windows”。
使用修改过的YaHei Mono字体并添加了特殊符号集。
使用Material Design风格的终端配色。

通过挂载远端媒体目录来与Windows交换文件(说是“交换”并不准确，因为文件本来就是存在远端的)，VirtualBox的默认共享目录在/media/<共享目录名>。
我将Workspace挂载到此并将之软连接到/works，这样就可以使用Vim或Emacs编辑HDL，再用Vivado综合。
NAS整个目录挂载到/medias。这样可以很方便的用Shell或Python处理文件(查重，批量解压/压缩/重命名，转码(ffmpeg))。

使用zsh作为Shell(并做了很多自定义的配置)。
在Emacs中安装evil，使用GDB调试，GCC编译。
Tcl可以连接到主机使用Vivado调试。
使用Python作为日常脚本。
使用org mode管理日程。
Gentoo的ebuild用得非常趁手。
Gentoo的“槽”特性对嵌入式来说简直好用，例如我有arm-none-llvm/gcc(Cortex M) arm-linux-(Cortex A) mips-linux-(Router) x64-linux-(Host)若干套编译器，一键升级，版本控制各种方便。

将操作系统(无论是Windows还是Linux的Virtual Disk)安装到SSD

你苹果爹就是不想你用垃圾HHD降低系统体验才用SSD坑你钱的

我平时使用根据个人习惯略改良过的80键键盘，在典型的应用中同时有chrome，smartPDF，altium designer/ssh/TeXLive分别对应绘制PCB，写代码(我在Linux下用vim作写作环境)和整理文章。
(在这些时候我左手从键盘左下角斜向上放着。无名指放在Tab上，大指拇放在ALT上)
这样除了方便切换各个窗口，浏览网页时按D来向下，打字时右手过来切到输入手势，绘制PCB时使用ALT快捷键也很有优势。我在空格键旁边增加了一个按键为ESC，这样就可以通过大拇指来输入ESC(同样的我也将Emacs的组合键大部分放在了空格旁边)。如果双手处在打字位置切换到浏览器后我会使用插件通过键盘控制浏览器。
当我需要任务栏时直接用meta键呼出meto菜单。而且meta键恰好在左手覆盖范围内。Windows在窗口最大化时无法呼出系统托盘，只要简单的使用opt键即可

我视线几乎不离开屏幕热区，就能完成我的大部分工作。
