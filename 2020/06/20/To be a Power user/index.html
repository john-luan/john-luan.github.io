如何使用计算机？

最近总是被人科普Windows不好用，macOS才是终极生产力工具balabala……
于是我想写一篇文章说说我是怎么使用Windows(指滚动升级并安装在一个还过得去的硬件上)的。

结论

苹果电子设备之间有良好的(Apple产品系内部)跨平台兼容性，可以给有必要使用苹果设备的用户提供连续生产力。
Apple的设备在“生产力”话题上，仅仅适用于极少部分影音工作者和为苹果开发这些行业软件的程序员。
而对于大多数人(硬件行业，机械行业，工业，科研，文字工作者)，其提供的生产力与Windows/Linux平台毫无可比性。

注意

我是一个果黑(显而易见)，如果问我原因，你会得到这个答案：

苹果的一些设备和服务的却不错，但在这家公司的逻辑中并没有对用户拥有该设备权力的尊重，它的用户们也并没有这样的需求；
其二，部分极端的果粉太过激进并精神入股了这家公司，毫无逻辑素养。
为了避免和这群人被同时称为‘苹果用户’我当前没有使用任何苹果的设备和服务

如果您是一位果粉(甚至比较狂热)，这篇文章极大可能将使您感到不适。
但显然，我并不是想在这个地方争论。
我甚至都没有开放评论，因为这个是我自己的页面，我在这个空间里享有绝对的言论自由。
您应该意识到这一点并为自己接下来的阅读状态负责。

选择软件的原则

避免国产管家类软件;
评估需求而选择软件，不要瞎装，污染环境;
不要带有偏见;
尽可能的使用快捷键;

为何打开电脑 & 需求

用网络来打发时间并不是一件好事，这篇文章也着重讨论“use”而非“play”。
坐到电脑前，就应该清楚“要做什么” 以及 “用什么工具做什么”。
如果对这个问题感到迷茫，可以试试用emacs的org-mode管理你的时间。
或者出去走走，运动或者泡咖啡馆什么的。你应当成为一个现充:)

  1. 我同时需要Windows和Linux环境，我的绝大部分生产力工具只在这两个平台上提供
  2. 我需要使用或者搭建云服务在各个设备间同步工程文档，所以我的本地系统需要一定程度上与它们可以互操作

我打开电脑做这么几件事：

  1. 电路设计(KiCAD/Altium/Cadence SPB)
  2. 一些关于FPGA/ARM/Linux的开发(gcc toolchain/vivado/yosys…)
  3. 使用PyQt编写GUI小工具
  4. 使用一些命令行下的工具使“使用计算机工作”这件事变得更容易
  5. 写文档，写博客
  6. 爱好的辅助：一些简单的音视频的工作
  7. 两个游戏：Starcraft II， cs go
  8. GTD(emacs)
  9. 一些琐碎的文档工作(OneNote/PowerPoint)

硬件

  • 一台作为”外出将就”使用和terminal的surface pro 5 i5/8G/256G
  • 一个VPS，某随时限速ban号的日本提供商
  • 一个All in One的台式主机，配置如下:

处理器

目前我使用i9 9900K处理器，超频至4.8G使用

运行内存

目前我计算机安装了32+16+16共64G DDR4 2133内存。在适当的时候，会添加一条32G同规格内存，达到4个插槽满载共96G。

显卡

目前我使用RX580作为游戏显卡。在适当的时候有计划升级为Tesla T4(不太确定)

存储

目前我有一个附加的磁盘柜，通过SFF-8088光纤连接器连接到主机。目前主板上安装有一条2TB的NVMe硬盘，一条2TB SATA硬盘。
阵列卡使用了Avago LSI MegaRAID SAS 9364

管理存储资料的原则：媒体资料存储在磁盘柜(NAS)上，代码同步到Git，代码外项目存到workspace，通过Onedrive同步。
及时整理媒体文件。如果拍了照，就及时选。如果有值得收藏的电影，就打好tag分门别类放好。
始终保持媒体或者数据是”干”的。

存储空间的分类

2T SSD - KVM Host/VM/Softwares
2T SSD - Workspace(工程目录) & host data ，该目录下子目录使用前缀进行区分
prj_ 软件项目
brd_ PCB项目
lrn_ 为学习某种技能/语言/工具设置的测试目录(例如lrn_cl = learn common lisp)
red_ 阅读目录，我读书有记笔记的习惯。混合使用Markdown, OneNote和Latex
cpl_ 一些需要自己构建/通过Git追新的软件的编译目录(例如vim，我总是编译最新版)
Windows Users Dir
2T Seagate ST1000LM035×5 - GPT - RAID6 - Media目录
通过KVM HOST统一管理的媒体目录

2T WESTDATA Blue SATA×5 - MBR - RAID0 - Media暂存
移动硬盘

系统架构

将操作系统(无论是Host还是VM)安装到SSD。使用VM从根本上杜绝了单独使用Linux作为工作系统时的文件同步，输入法，图形界面，Office，Flash，IE，网银等等等等问题。而且可以同时开启，方便的切换。正确的做法是把Linux和Windows融为一体，而不是不休止的争论他俩的好坏。Linux成为一个字符终端，一个编辑器，一个IDE，一个文件管理工具；而Windows成为一个图形终端，一个浏览器，一个Workbench。 (但是macOS做不到，它只能代替Linux或者与Windows非常尴尬的共存(使用mac不推荐的第三方引导安装双系统或浪费大量(尤其是(并不宽裕的)图形)性能给虚拟机中的Windows))。

  • 使用Debian作为VM主机，运行KVM，该系统上进行基本的VM管理工作和日志监控系统，不安装任何应用软件。使用xfce4作为DE，但进入系统后不使用Host桌面。
  • 在KVM上运行
    1. Gentoo(Embedded development)
    2. Debian(Router)
    3. nano-server(Professional Simulation Environment)
    4. Windows 10(Entertainment)
    5. Windows 10(Workstation)
  • 通过KVM管理磁盘并提供给各个操作系统远程挂载
  • VM上的Windows中不储存任何资料，仅仅安装系统和软件
  • 在Surface上运行Windows，并通过vnc或ssh连接到局域网内的VM或VPS。

    VMs

    Windows-workstation

  • 不安装第三方杀毒软件，管家软件。系统自带Defender & 防火墙够用了。
    感染病毒的可能性很低而且国产管家软件全家桶在我看来的作用仅仅是拖慢电脑运行速度和弹窗。
    (这就解释了为什么我的电脑没有弹窗)
  • 不安装im软件，对我来说它们在手机/Chrome上已足够用。(但是我装了个telegram的客户端，有时候用来传点东西挺方便的)
  • 系统自带输入法够用了。并且提供日文输入
  • 将所有配置，包括/user目录卸载到NAS上
  • 开机启动项
    • 一个开机启动的bat脚本 - 包含网络工具，一些快速恢复工作环境的基本配置
    • AHK - 为了在一些环境下使用快捷键
    • OneDrive - 用于多设备同步，云端备份
    • WOX - 做得比Microsoft Power toys还好的快速启动工具(真要感谢做得像屎一样的power toys)
    • Everything - 代替Windows索引服务(真要感谢做得像屎一样的windows search)
    • Ditto - 剪切板增强
    • QTranslate - 惭愧
    • Snipaste - 截屏增强
    • logi的鼠标工具 - 多设备切换必备

我几乎可以使用快捷键完成的功能都会使用快捷键(这也就解释了为什么我在除制图以外不使用鼠标和触摸板)。
没有快捷键的软件，我会尽量使用AHK和软件API使用快捷键。
我在Windows下有很多用起来非常舒服的软件。这些软件极大的提升了我的计算机使用效率。(见附录)

Windows-entertain

为打游戏而装的虚拟机，显示性能好像完全没有问题，变人中等分辨率能到25-30fps

  • 显卡直通
  • 通过USB声卡提供声音
  • 暴雪战网，steam
  • 最小化的额外配置

gentoo & debian

通过局域网或者DDNS访问。部分嵌入式的工作目录挂载到这里。我通过ssh(fluent terminal)连接到虚拟机，纯CLI全屏使用。
通过挂载媒体目录来与Windows交换文件(说是“交换”并不准确，因为文件是KVM统一管理并提供的)，
我将Workspace挂载到此并将之软连接到~/works，这样就可以使用Vim或Emacs编辑HDL，再用nano-server上的Vivado综合。
挂载~/medias。可以很方便的用Shell或Python处理文件(查重，批量解压/压缩/重命名，转码(ffmpeg，x265))。

  • 我使用zsh作为Shell。
  • 嵌入式c工程在Emacs中使用GDB调试，GCC编译。
  • 方便的使用例如Tcl等脚本环境
  • 使用Python作为日常脚本。

我的工作流

我平时使用日语配列的60+键蓝牙键盘，在典型的应用中同时有chrome，edge(PDF)，EDA软件/FEM软件/ssh/TeXLive来工作，写代码和整理文章。
(在这些时候我左手从键盘左下角斜向上放着。小指放在Ctrl上，无名指放在Tab上，大指拇放在ALT上)
这样除了方便切换各个窗口，浏览网页时使用vimium操作网页，打字时右手过来切到输入手势，绘制PCB时使用ALT快捷键也很有优势。我在空格键旁边增加了一个按键为ESC，这样就可以通过大拇指来输入ESC(同样的我也将Emacs的组合键大部分放在了空格旁边)。如果双手处在打字位置切换到浏览器后我会使用插件通过键盘控制浏览器。
当我需要任务栏时直接用meta键呼出meto菜单。而且meta键恰好在左手覆盖范围内。Windows在窗口最大化时无法呼出系统托盘，只要简单的使用opt键即可

我视线几乎不离开屏幕热区，就能完成我的大部分工作。

一些说明

  • 为什么把C:/User文件夹独立出来：
    这个文件夹下其实是有一些配置文件和软件临时目录的，既需要一定程度的访问速度，又不是存粹的软件。放出来管理比较有优势
  • 为什么不能是mac？
    mac 96G内存多少钱？装Debian+KVM有啥区别？装的上吗？能显卡直通打星际吗

反驳(持续更新)

  1. macOS没有广告
  2. macOS没有弹窗
    同学，早点参加工作吧，你们会议室里的电脑上如果不装360有毛的弹窗
    请搜索“如何关闭Mac上烦人的通知？”
  3. 学习macOS不需要学习注册表
    但是需要学习船新的“手势操作”，touchbar，访达文件管理器
  4. macOS不能玩游戏，所以能专心工作
    是你的问题，不是操作系统的问题
